## Summary (for Codex)

Apply when code involves **Memory, Record, Highlight, or date-based data**.

This rule defines core domain constraints.

---

## Rules

- Memory = exactly:
    - 1 image
    - 1 memo (String)
    - 1 score (Int, 1–10)
- Multiple memories per day allowed
- Exactly **one representative memory per day**
- Memory creation allowed **only in Present state**
- Past = read-only
- Future = goals only (no memories)

❌ Do NOT allow editing past memories

❌ Do NOT model memory as a list of images
